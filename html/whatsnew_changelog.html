• NEW
    Better support for list files.
    - asm (source) files and list files are now treated separately.
    - Labels/references between asm and list files are not mixed anymore.
    - The syntax highlighting for list files has been improved.
  • CHANGED
    The "asm-code-lens.includeFiles" setting was removed.
    It is now determined by the file association if the file should be included. Add all files you want asm-code-lens to recognize into the "Assembler file" (for asm files) or "Assembler list file" (for list files) file association.
  • CHANGED
    'Find Labels with no Reference' now only prints unused labels of the current workspace folder.
  • FIXED
    The problem matcher for sjasmplus ("$problem-matcher-sjasmplus") now supports both, absolute and relative, paths.
  • FIXED
    Multiroot: Each workspace folder and configuration works separately from the other. References from workspace folder A will not show up in workspace folder B.
